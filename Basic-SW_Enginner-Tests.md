# 学習内容

## 第一章：ハードウェア
- Unicode:単一のコード→世界各国のコード
    - UTF-8はUnicodeの符号化方式（文字の振り方）の一つ
- コンピュータの５大装置：記憶装置、制御装置、演算装置、入力装置、出力装置、
    - なぜか演算装置を覚えていなかったので気をつけて！
- 命令実行サイクル：命令フェッチ→命令の解読→実行アドレス計算→オペランド読出し→命令の実行→演算結果収納
    - 教科書P25に記載がある。全然覚えられる気がしない。
- ### アドレス指定：
    - 即値アドレス指定：命令のアドレス部に直接データを格納している方式
    - 直接アドレス指定；命令のアドレス部の値に実行アドレスを格納する方式
    - 間接アドレス指定：命令のアドレス部の値に「実行アドレスを格納しているアドレス」を格納する方式
        - 二重三重にアドレスが入子構造で格納されている場合もあるそうな。
    - 相対アドレス指定：命令アドレス部の値と命令アドレスレジスタ（プログラムカウンタ）の値を加算し、実行アドレスとする方式。
    - インデックスアドレス指定：命令アドレス部の値とインデックスレジスタの値を加算し、実行アドレスとする方式。
    - ベースアドレス指定：命令アドレス部の値とベースレジスタの値を加算し、実行アドレスとする方式。
- ### CPUの高速化
    - パイプライン処理：命令実行の各ステージを並行して実行することで高速化を図る
        - 豆知識：CISCとRISC
            - CISC:Complex Instruction Set Computer: PC用の複雑な命令体系
            - RISC:Reduced Instruction Set Computer: 組み込みシステムでよく使われる。
- ### 記憶素子
    - RAM:RandomAccessMemory: 読み書きできる。揮発性メモリ
    - ROM：ReadOnlyMemory： 読むだけ。不揮発性メモリ
    - RAMを詳しく！
        - DRAM:　Dynamic RAM: 後述のSRAMに比べて高集積化が容易→大容量で安価。**主記憶装置に使われる。**
        - SRAM:Static RAM: 高速だが、少量で高い（フリップフロップ回路の集積物）、キャッシュメモリに使われる。
    - ROMを詳しく！
        - マスクROM:製造時に書き込まれたものから変更できない
        - EPROM:紫外線照射でリセット可能
        - EEPROM:電圧をかけて部分消去できる
        - フラッシュメモリ：電圧をかけて任意に消去できる。
    - キャッシュメモリ：主記憶装置とCPUの間に入るメモリ。CPUが読出しが早いが、主記憶装置は遅いので、その性能差を保管するために挟まれる。一次、二次キャッシュと段階的になることも。
        - キャッシュメモリの書き込み方式
        - ライトスルー方式：速度通常で安定
        - ライトバック方式：高速だが、キャッシュが失われることもある。
    - ディスクキャッシュ：主記憶装置と磁気ディスクの間に配置し、高速化を図るための仕組み。
    - 記憶装置のアクセス速度：”れきしで”早い
        - レジスタ、キャッシュメモリ、主記憶装置、ディスクキャッシュ、ハードディスク。
    - メモリインタリーブ：主記憶装置を幾つかの区画に分割して、同時アクセスすることで高速化する仕組み
- ### 補助記憶装置
    - CD,DVD,BD,SDカードなど
        - CD:700MB
        - DVD:17GB
        - BD:128GB
        - SDカード:2GB
        - SDHC:32GB
        - SDXC:2TB
    - SSD:SolidStateDrive:フラッシュメモリで構成された記憶媒体。磁気ディスクの代わりに。
    - HDD:HardDiskDrive
        - 性能評価指標：アクセス時間
            - 位置決め時間、回転待ち時間、データ転送時間によって変動する。
        - フラグメンテーション（断片化）：データの書き込みと消去を繰り返す内に一つのディスクに情報が分散された状態。アクセスが非常に遅くなった状態だぞ！
- ### 入力装置
    - マウスやキーボードだけじゃなく、バーコードや、タッチパネル、ジョイステイック、OCRなど。
    - バーコードやPOSシステムが聞かれることが多いようだ。
    - POSシステムは同一商品の購入頻度を測定的、重要なマーケティングツールになっている。
- ### 出力装置
    - ディスプレイ関連に解像度やデータ量を問われる問題が出たりする。
        - データ量を考慮する時には、byteとbitで間違えないようにしたい。
    - 他にプリンタ関連で、３Dプリンタが出力されたり、最近のトレンドを捉えたものが出ているようだ。
- 入出力インターフェース
    - USB,シリアルATA、HDMI,DVIーI(29ピンとかの白い古めのプラグ）
    - 他にIrDA（赤外線通信企画）、BlueTooth、Zigbee（近距離無線技術、低コスト、低出力）
        - 2.4Ghz帯は無線免許で誰でも取り扱うことができるところだ。

## 第二章：ソフトウェアとOSS
- ソフトウェアの分類
    - システムソフトウェア
        - ミドルウェア
        - 基本ソフトウェア（OSなど）
            - 制御プログラム
            - 言語プロセッサ
            - サービスプログラム
    - 応用ソフトウェア
        - アプリケーションソフトウェアともいう。大体のソフトウェア開発。
- スループット：単位時間あたりの処理速度
    - MIPS値：１秒間あたりに実行される命令の回数を百万単位で表したもの
- 信頼性の向上：評価基準はRASIS
    - RASIS：信頼性、可用性、保守性、保全生、安全性の頭文字をとった略語
        - Reliabitlity,Availability,Servieability,Integrity,Security
- ジョブ管理とタスク管理
    - ジョブ：利用者（ユーザー）から見た仕事の単位
    - タスク：PCから見た時の仕事の単位
        - タスクはPCの中で状態が以下の三つのように変遷しながら消化される。
            - 実行可能状態
            - 実行状態
            - 待機状態
        - ディスパッチャ：タスクの中から次に実行可能なタスクにCPUの使用権を与える処理（ディスパッチ）を行う処理部
            - 優先度方式：優先度の高いタスクから実行する
            - ラウンドロビン方式：CPUの処理時間を微小に分割し、一定の時間処理したタスクから次のタスクに処理を写していく方式。
- スプーリング：入出力データを一時的に磁気ディスク装置などに蓄える処理　→　CPUの有効活用に繋がる。
- マルチプログラミング＝マルチタスク：ある処理における入出力処理待ち時間等で、CPUの処理の停止した際に、別の処理を実施させる方式。主記憶上に複数のタスクを置いて実現する。
    - プリエンティブ：OSがタスクにCPUの使用権を与える方式。現在の主流。タスクが自発的にCPUを解放する形式がノンプリエンティブ
- 記憶管理
    - 実記憶管理：主記憶のどの場所にどのようなプログラムを読み込んでおくかの管理。
        - プログラム記憶方式はプログラムを主記憶に読み込んでおき、実行時にそのプログラムをロード、実行の完了時にプログラムを破棄（解放）する処理
            - 固定区画方式と可変区画方式がある。固定区画方式では余った領域を使用することはできない融通の無さがある。可変区画方式は以下のフラグメンテーションの課題が存在する。
            - 可変区画方式では当然ながらメモリがフラグメンテーションする。ので、その対応のために、「ガーベージコレクション」や「メモリコンパクション」を行う。
            - メモリリーク：バグなどで、動作中に主記憶に確保したプログラムが解放されない状態。
        - オーバーレイ方式：プログラムを同時に実行しない排他的ないくつかの単位であるセグメントに分類しておき、実行時に必要な部分だけをロードして実行する方式。大きなプログラムを小容量の主記憶上で実行させる手段として利用される。
    - 仮想記憶管理
        - 高速で小容量の主記憶（実記憶）と主記憶に比べて低速で大容量の磁気ディスク（仮装記憶）を組み合わせ、見かけ上、高速で大容量のアドレス空間を提供するシステムのことで、ほとんどのOSで採用されているシステムがこれに当たる。
            - DAT（動的アドレス変換機構）：仮想記憶管理ではアドレスが仮想アドレスで扱われるので、主記憶上にロードされる際に、実アドレスへの変換が必要になる。この時に使われる変換形態の内の一つがDAT。
        - ページング方式：仮想記憶管理での代表的な方式。（P79に詳しい）
            - **くればで覚える** :ページング方式はページ（固定長単位での管理。ページング（後述）が多発する->処理効率定価->スラッシング
                - ページング：実行に必要なページを補助記憶から主記憶上にロードする処理（ページイン）と実行に必要のないページを補助記憶に退避する処理（ページアウト）の二つの処理の総称。ページスワッピングとも言う。
                - スラッシング：ページングに伴う処理速度の減少のこと。
            - ページ置き換えアルゴリズム：（マニアックすぎる内容かな…）
                - FIFO:一番古くから存在するページを置換
                - LRU:最後に参照されてからの経過時間がもっとも長いものを置換
                - LFU：参照回数の最も少ないものを置換。
